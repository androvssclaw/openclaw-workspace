# VPN Anti-Block Runbook (AmneziaWG) — 5 минут

Цель: быстро восстановить доступ, если текущий профиль начал блокироваться.

## Принцип
Меняем не всё сразу, а по шагам: порт → профиль/ключи → параметры обфускации.

## Шаг 0 — Быстрая диагностика (read-only)
```bash
./scripts/vpn_status.sh
openclaw status
```

## Шаг 1 — Быстрый fallback (самый дешёвый шаг)
1. Переключиться на заранее подготовленный резервный профиль.
2. Проверить подключение 1–2 минуты.

Если не помогло — шаг 2.

## Шаг 2 — Ротация порта
1. Выбрать новый UDP порт (предпочтительно до 9999, не занятый сервисами).
2. Применить порт в настройках AWG.
3. Пересоздать/обновить клиентские конфиги.
4. Проверить подключение.

Если не помогло — шаг 3.

## Шаг 3 — Ротация ключей/конфигов
1. Выпустить новые ключи/конфиги для клиентов.
2. Старые ключи отозвать.
3. Проверить handshake и трафик.

Если не помогло — шаг 4.

## Шаг 4 — Смена параметров обфускации
1. Обновить I-параметры/сигнатуру обфускации (по официальной документации).
2. Выпустить новые клиентские конфиги.
3. Проверить подключение.

## Операционные правила
- Изменения firewall/docker/systemd/сетевых настроек — только с подтверждением Andrey.
- Всегда менять по одному фактору за итерацию, чтобы понимать, что сработало.
- Держать минимум 2 резервных клиентских профиля заранее.

## Мини-чеклист после восстановления
- [ ] Подключение стабильно 10+ минут
- [ ] Проверен доступ к нужным сайтам/сервисам
- [ ] Обновлены рабочие клиентские профили
- [ ] Зафиксированы изменения в README/TASKS
